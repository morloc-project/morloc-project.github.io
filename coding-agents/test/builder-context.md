All projects should:
 * Have comprehensive docstrings on all exported functions (descriptions, arg
   names, defaults, literal flags, unrolled records)
 * Avoid superficial morloc implementations

In every project, explore as many advanced morloc features as possible:

 * getter/setter patterns
 * infix operator definitions
 * recursion
 * arrow tables
 * tensors
 * effects and delayed evaluation
 * creative uses of intrinsics
 * typeclasses
 * type hierarchies

Explore the edges of what is possible with morloc. Explore the interaction
between these features.

Do not be passive, think creatively and wildly about what should be possible. Is
the syntax limiting? Are there things you want to be able to do that the
language makes difficult?

If you find a bug, unexpected behavior or bad error messages, this is GREAT
SUCCESS! When you find a bug, drop everything you are doing and write a
dedicated finding at findings/<project-id>/finding-NNN.md (where NNN is an
integer) using the finding format from your agent instructions.
